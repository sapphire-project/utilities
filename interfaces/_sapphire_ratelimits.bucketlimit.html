maximum

maximum: number
  • Defined in ratelimits/src/lib/Bucket.ts:39

The maximum amount of tickets.

This limits the amount of requests that can be made within the Bucket's {@link Bucket#delay delay}.

timespan

timespan: number
  • Defined in ratelimits/src/lib/Bucket.ts:31

The time between tickets.

If timespan is zero, there will be no spacing between usages, this means that a 5/5s bucket ({@link BucketLimit#limit limit} of 5 in 5 seconds {@link Bucket#delay delay}) will accept 5 requests, then will wait for the remaining time.

However, if in the previous example, this is set to 1 second, this will then space the requests evenly to 1 request per second until the bucket is consumed.
